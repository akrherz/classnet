Assignment Development

A Test meta language is useful in uploading and downloading assignments since the assignment can be built with a text editor. Example:
 
<TEST>
…HTML…
<Q>
The acronym HTML stand for<BR>
<MC PTS=1> HyperText Markup Language
<MC> HypoText Markup Language
<MC> HyperText Makeup Language
</Q>
</TEST>
 

This meta language can be described using the Extended Meta Language (XML) being developed by the W3C.  Here is a portion of the document type definition (DTD) for the Test meta language:
 

<!ELEMENT Q  (MC |CHECK| BLANK| ESSAY| OPTION| LIST| %body.content;)+>
<!ELEMENT MC EMPTY>
<!ATTLIST MC
    PTS CDATA "0"
    BR (yes|no) "yes">
<!ELEMENT CHECK EMPTY>
<!ELEMENT BLANK (#PCDATA)>
<!ELEMENT ESSAY (#PCDATA)>
<!ELEMENT ITEM (#PCDATA)>
<!ELEMENT OPTION (ITEM)+>
<!ELEMENT LIST (ITEM)+>
<!ELEMENT MQ (Q)+>

Using this definition and a freely available XML parser, a test can be read, processed and written by the server. By defining a Calendar DTD, the same technique can be used to process institutional or class events. In fact,

this technique provides a general mechanism for processing instructor supplied data.
