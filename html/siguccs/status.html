Status of Current System

A success
  • 99 Classes, 2286 students
  • Written in Perl
  • Run on a Unix DecStation 5000/125 and DecStation 5000/133
  • 25,000 accesses/week
  • Includes students at University of Minnesota, University of Northern Alabama and state high schools
  • Used for practice tests, weather forecasts (250 students several times a week!), essay tests, proctored exams, surveys, simulations, discussions, personality testing, student evaluations, teacher training...
  • Most management tasks are automated (backups, enrollment and expiration of accounts and classes)
but... 
  • As the functionality grows the Perl code gets slower
  • Every menu access is a server hit
  • Using HTML forms limits interactivity
  • Underlying code has become difficult to manage
Why bother?

Aren't there other systems that do this?

There are a number of classroom management tools (see the review by the University of Iowa). These tools are usually used for content preparation and delivery with some assignment evaluation and communications included.

We chose to focus on the management of assignments for several reasons:

  • There are already many tools for Web page development and content delivery
  • Most management tools impose a structure on the content which instructors quickly find constraining
  • Many tools store the content on their server concentrating access
  • Assignments are at the heart of the learning process
