Adding Assignments

  • Select the type of assignment
  • Fill in the assignment name (15 character limit).
  • Select whether the assignment will be immediately available to students, or if it will be stored in a development folder and published later.
  • If you have already prepared the assignment, you may upload it at this time.
    • If the file is stored in a WWW directory, supply the URL in the Initial URL field.
    • If the file is stored on your computer, use the Browse button to locate and select the file.
  • Click on Add. If you do not Add the assignment, it will not be available for editing or for student use.

  • When students submit their assignments, ClassNet will grade them according to the answers you have given.

  • Reset will clear all data previously entered in the form.

Assignment Types

ClassNet is designed to support a variety of assignment types. If you have suggestions for assignment types, let us know.

  • TEST - A test is a series of multiple-choice, fill-in-the-blank and essay questions. Random versions of tests can be administered, or individual questions can be randomly generated. When students submit their tests, the multiple-choice and fill-in-the-blank questions are automatically graded. Essay questions are e-mailed to the instructor for evaluation and comment.

  • FORECAST - A forecast is a test which requests information about tomorrow's expected weather at different sites in the U.S. Students may complete forecasts on a daily basis.

  • INCLASS - An in-class assignment provides the ability to record scores from tests or assignments which are not administered through ClassNet. Instructors create an in-class assignment, and then enter student scores. These scores will be reported along with the other assignment types.

  • EVAL - A student evaluation form is a test with a due date. Students may enter and edit their answers until the due date. Instructors may view summary results after the due date but cannot view individual student responses. Essay responses may be viewed by using the Data option.

  • DIALOG - A dialog assignment allows for private, ongoing conversations between instructors and students. These conversations are stored by ClassNet, and can be analyzed and synthesized for research purposes at a later date. This assignment is ideal for students in areas such as teacher education and counselor education who wish to hold practice conversations with secondary/elementary students/clients.

  • JAVA - A Java assignment is a more complicated assignment type that stores Java simulation data. This allows instructors to analyze how students are working with simulations. This type of assignment involves designing a Java applet which interacts with a second data server. This assignment type is a powerful tool, and needs to be developed with the ClassNet administrators. However, it is possible to create a simple Java assignment that does not store data on the ClassNet server. This does not require consultation with ClassNet administrators, and allows students to be given access to an assignment containing a Java applet.
